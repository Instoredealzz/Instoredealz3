## Creating Your First Deal

### Deal Planning
Before creating a deal, consider:
- **Target Audience**: Who are your ideal customers?
- **Profit Margins**: Ensure profitability after discount
- **Inventory**: Confirm product availability
- **Timeline**: Set realistic validity periods

### Step-by-Step Deal Creation

#### 1. Access Deal Creation
- Login to your vendor dashboard
- Click "Create New Deal"
- Select deal type (percentage discount, fixed amount, buy-one-get-one)

#### 2. Basic Information
- **Deal Title**: Clear, attractive name (e.g., "50% Off Designer Clothing")
- **Description**: Detailed explanation of what's included
- **Category**: Select appropriate business category
- **Subcategory**: More specific classification

#### 3. Discount Details
- **Discount Type**: 
  - Percentage (e.g., 25% off)
  - Fixed amount (e.g., ₹500 off)
  - Buy X Get Y (e.g., Buy 2 Get 1 Free)
- **Original Price**: Pre-discount price
- **Final Price**: Price after discount
- **Minimum Purchase**: Set minimum order value (optional)

#### 4. Validity & Limits
- **Start Date**: When deal becomes active
- **End Date**: Deal expiration
- **Daily Limits**: Maximum redemptions per day
- **Total Limits**: Maximum total redemptions
- **Per Customer Limit**: How many times one customer can claim

#### 5. Images & Media
- **Main Image**: High-quality product/service photo
- **Additional Images**: Gallery of related photos
- **Video**: Product demonstration (optional)

**Image Guidelines:**
- Resolution: 1200x800 pixels minimum
- Format: JPG or PNG
- Size: Under 5MB per image
- Show product clearly
- Good lighting and composition

#### 6. Terms & Conditions
- Specific usage terms
- Exclusions (what's not included)
- Special requirements
- Refund/exchange policy

#### 7. Location Settings
- **Single Location**: One store address
- **Multiple Locations**: Add all participating stores
- **Service Area**: Delivery/service radius (if applicable)

### Deal Approval Process
1. **Submission**: Deal sent for review
2. **Automated Check**: System validation (instant)
3. **Admin Review**: Quality and compliance check (2-24 hours)
4. **Feedback**: Approval or revision requests
5. **Go Live**: Deal appears on platform

## POS System Integration

### Setting Up Your POS Terminal

#### Hardware Requirements
- Tablet or computer with internet
- Receipt printer (recommended)
- Barcode scanner (optional)
- Credit card reader (if accepting cards)

#### Software Setup
1. **Login**: Use your vendor credentials
2. **Terminal Registration**: Register each POS device
3. **Configuration**: Set up printing, payment options
4. **Testing**: Process test transactions

### API Keys for Third-Party Integration

If you use a third-party POS system and want to integrate it with Instoredealz for automated claim verification, you'll need an API key.

#### Viewing Your API Keys

1. **Access API Keys Page**: Navigate to `/vendor/api-keys` from your vendor dashboard
2. **View Key Details**: See all your API keys with:
   - API Key (for authentication)
   - Status (Active/Inactive/Expired)
   - Creation date
   - Last used date
   - Rate limits
   - Expiration date (if applicable)

#### Receiving Your API Key

When an administrator generates an API key for your business:

1. **Email Notification**: You'll receive an automated email with:
   - Your business name
   - The complete API key
   - Creation date and rate limit information
   - Security best practices
   - Usage instructions

2. **Dashboard Access**: You can view your API keys anytime from your vendor dashboard under "API Keys" menu

3. **Copy to Clipboard**: Use the copy button to easily copy your API key for integration

#### Using Your API Key

**Security Best Practices:**
- ⚠️ Never share your API key publicly
- Keep it secure in your POS system's configuration
- Don't commit API keys to version control (Git, etc.)
- Use HTTPS for all API requests

**Integration Example:**
```bash
# Example API request with your key
curl -X POST https://api.instoredealz.com/v1/claims/verify \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"claimCode": "ABC123"}'
```

**Rate Limits:**
- Monitor your API usage to stay within rate limits
- Default: 1000 requests per hour
- Contact support if you need higher limits

**Need Help?**
- Contact support for API documentation
- Request integration assistance
- Report any API-related issues

### Daily Operations

#### Starting Your POS Session
1. Login to POS system
2. Select "Start New Session"
3. Verify connection status
4. Check deal availability
5. Ensure receipt printer is working

#### Processing Customer Claims

**Method 1: QR Code Scanning**
1. Customer shows QR code from app
2. Scan with POS system camera
3. System displays deal details
4. Verify customer identity
5. Process transaction

**Method 2: Manual Code Entry**
1. Customer provides 6-digit claim code
2. Enter code in POS system
3. System validates code
4. Confirm deal details
5. Process transaction

**Method 3: Phone/Name Lookup**
1. Ask for customer phone number or name
2. Search in POS system
3. Select correct customer
4. Choose from their active claims
5. Process transaction

#### Transaction Steps
1. **Claim Verification**: Validate customer code
2. **Deal Application**: Apply discount automatically
3. **Payment Processing**: 
   - Calculate final amount
   - Accept payment (cash/card)
   - Generate receipt
4. **Completion**: Mark claim as used

### Handling Transaction Issues

**Invalid Codes:**
- Double-check code entry
- Verify deal validity dates
- Check if already used
- Contact customer support if needed

**System Offline:**
- Record transaction details manually
- Process when connection restored
- Keep receipt copies for verification

**Customer Disputes:**
- Review transaction history
- Check claim details
- Document the issue
- Escalate to support if needed

## Troubleshooting

### Common Issues and Solutions

#### Login Problems
**Symptom**: Cannot access vendor dashboard
**Solutions**:
- Check internet connection
- Clear browser cache and cookies
- Try different browser
- Reset password if needed
- Contact support for account issues

#### Deal Approval Delays
**Symptom**: Deal pending approval for >48 hours
**Solutions**:
- Check for missing information
- Verify all documents uploaded correctly
- Review compliance with platform guidelines
- Contact admin support with deal ID

#### POS System Issues
**Symptom**: Cannot process transactions
**Solutions**:
- Check internet connectivity
- Restart POS application
- Verify login credentials
- Clear application cache
- Switch to manual processing if needed

#### Payment Settlement Delays
**Symptom**: Expected payment not received
**Solutions**:
- Check bank account details
- Verify transaction completion
- Review settlement schedule
- Contact finance team
- Provide transaction references

#### Customer Complaints
**Symptom**: Unhappy customers
**Solutions**:
- Listen to customer concerns
- Review transaction details
- Check deal terms compliance
- Offer appropriate resolution
- Document case for future reference

### Error Code Reference

#### Common Error Codes
- **E001**: Invalid claim code
- **E002**: Expired deal
- **E003**: Code already used
- **E004**: System connection error
- **E005**: Insufficient inventory
- **E006**: Payment processing failed
- **E007**: Customer verification failed

### Emergency Procedures

#### System Downtime
1. **Document Transactions**: Write down details manually
2. **Inform Customers**: Explain temporary issues
3. **Alternative Processing**: Use backup methods
4. **Sync When Online**: Upload manual transactions
5. **Report Issues**: Contact technical support

#### Suspicious Activity
1. **Document Everything**: Screenshots, details
2. **Stop Processing**: Temporarily halt transactions
3. **Contact Security**: Report immediately
4. **Preserve Evidence**: Don't modify anything
5. **Follow Guidelines**: Wait for instructions

## FAQ

### Getting Started

**Q: How long does vendor approval take?**
A: Typically 24-48 hours after submitting complete documentation. Complex cases may take up to 5 business days.

**Q: Is there a fee to join the platform?**
A: No joining fee. We charge a small commission only on successful transactions.

**Q: Can I create deals for services, not just products?**
A: Yes! Service businesses like restaurants, salons, spas, and consultancies can create deals.

**Q: Do I need special equipment for POS integration?**
A: Just a device with internet access. Additional equipment like printers and scanners are recommended but optional.

### Deal Management

**Q: How many deals can I create?**
A: No limit on deal creation. You can have multiple active deals simultaneously.

**Q: Can I modify deals after they're approved?**
A: Limited modifications allowed. Major changes require new approval.

**Q: What if I want to cancel a deal early?**
A: You can deactivate deals, but existing customer claims remain valid.

**Q: How do I handle customer complaints about deals?**
A: Document the issue and work with our support team for resolution.

### Technical Issues

**Q: What if my internet connection fails during a transaction?**
A: Document the transaction manually and sync when connection returns.

**Q: How do I verify customers without the POS system?**
A: Use manual verification methods and update the system later.

**Q: Can customers use deals at all my store locations?**
A: Yes, if you specify multiple locations when creating the deal.

### Payments and Settlements

**Q: When do I receive payment for transactions?**
A: Settlements occur within 48 hours of transaction completion.

**Q: What are the platform fees?**
A: Competitive commission structure, detailed in your vendor agreement.

**Q: How are refunds handled?**
A: Platform facilitates refunds through original payment method.

**Q: Can I see detailed transaction reports?**
A: Yes, comprehensive reports available in your vendor dashboard.
